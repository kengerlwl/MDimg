# 应用的场景

有若干服务需要访问，他们或者ip不一样，或者端口不一样。

如果一个个的去绑定隐形url域名挺麻烦的。也没必要。

一个优秀的办法是，通过不同的域名访问过去。然后根据域名不同做反向代理。



# demo

我服务器上有一个wordpress，其端口是8081。我想要通过`blog.kenger.com`去访问该服务。

先直接将域名指向服务器ip。或者服务器www域名也可以。总之就是直接到80端口。
